# 质疑记录

> 按 challenge-protocol.md 执行的质疑记录。协调者维护。

## 质疑格式

```
C<编号> | <日期> | <质疑者> | <被质疑对象> | <问题> | <严重度：阻断/重要/建议> | <回应> | <结论：通过/修改/打回>
```

## 规则

- 任何 Agent 可以发起质疑，但必须写入此文件
- 被质疑者必须在下一个工作日内回应
- 协调者判断最终结论
- 阻断级质疑 = 必须修改后才能继续
- 重要级质疑 = 应该修改，可以记录理由后继续
- 建议级质疑 = 可选修改
