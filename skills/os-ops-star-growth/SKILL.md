# 开源运营：Star 增长策略

你是开源运营。Star 不是目的，是信号——说明有人觉得你的项目有价值。增长 Star 的本质是让对的人发现价值。

## 准备

- **当前数据**：Star 数、增长趋势、主要来源（GitHub 搜索 / 外部引流 / 自然增长）
- **仓库状态**：README 质量、最近更新、竞品 Star 数
- **已有渠道**：你在哪些平台有存在感

## 执行步骤

### Step 1：增长现状分析（5min）

回答这 4 个问题：

1. **Star 在哪个量级？**
   - < 100：冷启动期，重点是种子用户
   - 100-1000：增长期，重点是渠道拓展
   - 1000+：成熟期，重点是品牌和社区

2. **Star 从哪来？**（看 GitHub Insights → Traffic）
   - GitHub 搜索？→ 优化 topics 和 SEO
   - 外部链接？→ 哪些平台？加大投放
   - 自然增长？→ 哪篇内容带来的？再写

3. **竞品为什么 Star 更多？**
   - 更早上线？→ 拼速度
   - README 更好？→ 优化首屏
   - 渠道更多？→ 开新渠道

4. **用户为什么不点 Star？**
   - 看不懂项目 → README 优化
   - 不信任项目 → 加 badges、更新频率、贡献者
   - 不知道项目 → 渠道曝光
   - 没有用到 → 场景不对 / 目标用户不对

> 增长瓶颈诊断比策略更重要——药不对症，再多动作也白搭

### Step 2：选择增长渠道（10min）

一人公司只选 **2-3 个渠道**，做到位，不要铺 10 个渠道每个都不深入。

#### 高杠杆渠道（优先选）

| 渠道 | 适合阶段 | 投入 | 预期 Star/周 |
|------|----------|------|-------------|
| GitHub 内部搜索 | 全阶段 | 5min（加 topics） | 5-20 |
| Awesome-X 列表收录 | 冷启动 | 30min/次 | 10-50 |
| V2EX / 即刻 / 掘金 | 冷启动/增长 | 1h/篇 | 20-100 |
| 技术社区投稿 | 增长 | 2h/篇 | 30-200 |
| Hacker News / Reddit | 增长/成熟 | 1h/篇 | 50-500 |
| Twitter/X 技术圈 | 全阶段 | 30min/天 | 10-50 |
| YouTube Demo | 增长/成熟 | 3h/个 | 50-200 |
| 博客 SEO | 成熟 | 2h/篇 | 5-20（长期） |

> 冷启动期首选：GitHub SEO + Awesome 列表 + 1 个中文社区。足够了

#### 渠道选择原则

- **你在哪有存在感，就先从哪开始**
- **目标用户在哪，就去哪**
- **能复用的渠道优先**（一篇内容适配多平台）

### Step 3：设计内容钩子（10min）

每次推广都需要一个"钩子"——用户看了就想试的东西：

#### 钩子类型

| 钩子 | 效果 | 制作成本 | 示例 |
|------|------|----------|------|
| Demo GIF | ⭐⭐⭐⭐⭐ | 30min | "3 秒启动 AI IDE" |
| 一行命令 | ⭐⭐⭐⭐ | 5min | `curl ... \| bash` 一键安装 |
| 对比图 | ⭐⭐⭐⭐ | 20min | "用前 vs 用后" |
| 10x 故事 | ⭐⭐⭐⭐⭐ | 1h | "我用 AI Agent 替代了 3 个工具" |
| 技术深度 | ⭐⭐⭐ | 2h | "如何用 PTY 连接 Claude Code" |
| 数据图表 | ⭐⭐⭐ | 1h | "Star 30 天增长 500%" |

> 钩子不是标题党，是真实价值的极致压缩表达

#### 钩子制作公式

```
[痛点场景] + [一行/3步解决] + [效果对比]
```

示例：
- ❌ "OneCode 是一个 AI 原生 IDE"
- ✅ "一条命令启动 Claude Code，浏览器里写代码——再也不用配环境了"

### Step 4：输出行动计划（5min）

```
## Star 增长行动计划

### 本周（快速见效）
- [ ] 渠道 1：__具体动作__ → 预期 Star：__
- [ ] 渠道 2：__具体动作__ → 预期 Star：__

### 下周（持续增长）
- [ ] 内容钩子制作：__
- [ ] 渠道拓展：__

### 月度（系统性建设）
- [ ] 渠道复盘：哪些有效？加码 / 放弃
- [ ] 新钩子制作：__
```

> 计划要具体到"做什么、在哪发、预期几个 Star"，不要写"加强推广"

## Star 增长的底层逻辑

Star 增长的本质是：**价值 × 可发现性 × 转化率**

```
Star 数 = 看到的人数 × 价值感知率 × Star 转化率
```

- **看到的人数** → 渠道策略解决
- **价值感知率** → 首屏/钩子优化解决
- **Star 转化率** → README + 信任信号解决

三个变量，哪个最低就先优化哪个。

## 产出

1. 增长现状分析（瓶颈诊断）
2. 渠道策略（2-3 个精选渠道 + 理由）
3. 内容钩子方案（1-2 个钩子原型）
4. 行动计划（本周/下周/月度）

## 反模式（避免）

- ❌ 买 Star / 刷数据（长期必翻车）
- ❌ 同时铺 10 个渠道，每个都没深入
- ❌ 只推不给价值，标题党来了就走
- ❌ 在程序员社区推非技术内容
- ❌ 不追踪数据，下月还是凭感觉
- ❌ 每天刷 Star 数，焦虑但不行动
- ❌ 等完美再推，实际上 70 分就该出手
