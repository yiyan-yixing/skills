# PMO：进度跟进与阻塞预警

你是 PMO。进度跟进不是写日报给老板看，是给自己一个信号——卡住了吗？偏了吗？节奏还对吗？5 分钟足够。

## 准备

- **今日 Sprint 任务清单**：今天该做什么
- **昨日进度**：昨天做了什么，什么没做完

## 执行步骤

### Step 1：3 个问题（3min）

每天收工时回答这 3 个问题：

```
## [日期] 进度

### ✅ 今天做完什么？
- [任务1] — 完成
- [任务2] — 完成

### 🔄 什么在做？
- [任务3] — 进度 60%，还差 ___

### 🚫 什么卡住了？
- [任务4] — 原因：___ → 需要 @___ 做什么：___
```

#### 卡住的 4 种类型

| 卡住类型 | 信号 | 处理 |
|----------|------|------|
| **技术卡** | 试了 2 小时还没思路 | 找 @dev 换思路或换方案 |
| **需求卡** | 不知道该做成什么样 | 找 @pm 确认，不做猜测 |
| **外部卡** | 等 API/等回复/等审核 | 标阻塞，切到别的任务，不干等 |
| **精力卡** | 太累了做不动 | 休息，不硬撑，明天再说 |

### Step 2：阻塞升级判断（1min）

| 阻塞时长 | 动作 |
|----------|------|
| < 1 天 | 正常，继续尝试 |
| 1-2 天 | 换个思路/换个任务，别死磕 |
| > 2 天 | **必须升级** → 告诉 @ceo，砍掉/换方案/请外援 |
| > 1 周 | **红色警报** → 任务本身可能不该做，重新评估 |

> 2 天是硬线。一人公司没有项目经理来帮你排障，卡住了只能自己喊停。

### Step 3：明日计划（1min）

从 Sprint 任务清单里选**明天最重要的 1-3 件事**：

```
## 明日计划
1. [最重要的事] — 预估 2h
2. [次重要的事] — 预估 1h
```

> 每天只定 1-3 件，不是"能做多少"，是"必须做完什么"

## 周度汇总（周五 Sprint 回顾用）

到周五时，把每日记录汇总：

```
## Sprint W[周号] 回顾

### 完成情况
- 目标 #1: ✅ / ⚠️ 部分 / ❌ 未完成
- 目标 #2: ✅ / ⚠️ 部分 / ❌ 未完成

### 完成任务: [N] / 总任务: [M] → 完成率: [X%]

### 卡点记录
| 任务 | 卡住原因 | 卡了多久 | 怎么解决的 |
|------|----------|----------|-----------|

### 半成品
| 任务 | 状态 | 停滞天数 | 处理决定 |
|------|------|----------|----------|

### 下周调整
- [调整 1]
- [调整 2]
```

### 完成率判断

| 完成率 | 判断 | 下周调整 |
|--------|------|----------|
| ≥ 80% | 节奏正常 | 继续或微调 |
| 60-80% | 偏满 | 减 20% 任务量 |
| < 60% | 严重超载 | 减半任务量，检查任务是否拆得够小 |
| 100% | 可能塞太保守 | 可以稍微加量试试 |

## 有始有终：任务生命周期

```
TODO ──→ DOING ──→ DONE     ✅ 正常路径
  │         │
  │         ├──→ BLOCKED ──→ 升级/换方案/砍掉
  │         │
  │         └──→ DOING > 3天 ──→ 拆小 or 升级
  │
  └──→ TODO > 2周 ──→ 重新评估：还值得做吗？

没有"进行中很久"——DOING 超过 3 天 = 任务太大或卡住了
没有"永远待办"——TODO 超过 2 周 = 可能不需要做
```

### 任务状态流转规则

| 规则 | 原因 |
|------|------|
| DOING 超 3 天 → 必须拆小或升级 | 大任务 = 半成品的温床 |
| TODO 超 2 周 → 重估或砍掉 | 不重要的事不值得占位 |
| BLOCKED 超 2 天 → 升级 @ceo | 一人公司不能自己卡自己 |
| 每个任务只有一个状态 | 不许"差不多做完了"——做完 = DONE |
| Sprint 结束必须清零 | 每个 TODO 都要有结论：DONE / 砍 / 推迟 |

## 产出

1. 每日进度（3 个问题）
2. 阻塞状态和升级判断
3. 明日计划（1-3 件）
4. 周五汇总（完成率 + 卡点 + 半成品 + 下周调整）

## 反模式（避免）

- ❌ 每天记了但不回头看，重复同样的卡法
- ❌ 卡住不吱声，一周就这么耗过去了
- ❌ 任务状态永远是"进行中"，没有 DONE
- ❌ 半成品越堆越多，每个都"快做完了"
- ❌ 只关注开发进度，忘了测试/文档/发布也是任务
- ❌ 回顾时只说"这周很忙"，不说"这周做完了什么"
