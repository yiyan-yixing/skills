# 一人公司 Agent 协作流程

> 9 个角色，1 条产品闭环，极致快速地从想法到数据验证。

---

## 产品闭环流程

```
@pm 定义需求 → @designer 出原型 → @dev 开发 → @qa 测试
  → @devops 部署 → @ops 推送用户 → @data 收集效果 → @pm 决策迭代
                                                                ↓
                           @ceo 战略校准 ← @data 效果报告 ←─────┘
```

每个环节都有明确的输入、输出和交接标准。

---

## 角色职责速查

| 角色 | 调用 | 做什么 | 不做什么 |
|------|------|--------|----------|
| **CEO** | `@ceo` | 定方向、做决策、监控全局 | 不陷入执行细节 |
| **PM** | `@pm` | 定义需求、排优先级、写 PRD | 不自己当用户 |
| **Designer** | `@designer` | 快速原型、UI/UX、视觉冲击 | 不追求完美拖延交付 |
| **Dev** | `@dev` | 写代码、架构、修 bug | 不自己测自己的代码 |
| **QA** | `@qa` | 测试、找 bug、发布阻断 | 不证明代码没问题 |
| **DevOps** | `@devops` | CI/CD、部署、监控、工具链 | 不自建不需要的基础设施 |
| **Ops** | `@ops` | 推广、内容、增长 | 不追热点偏离定位 |
| **Data** | `@data` | 埋点、分析、效果报告 | 不堆数据不说结论 |
| **Fin** | `@fin` | 记账、现金流、支出审查 | 不混个人和公司支出 |

## 快速出 Demo 流程（3 天）

> AI Agent 赛道的铁律：没有 Demo 就没有对话。

```
Day 1: @pm 定场景 + @designer 画关键流程
       输入：市场假设 + 技术可行性
       输出：1 页 PRD + 核心交互原型

Day 2: @dev 快速实现 + @designer UI 走查
       输入：PRD + 交互原型
       输出：可运行 Demo

Day 3: @devops 一键部署 + @ops 推给 10 人看 + @data 开始采集
       输入：可运行 Demo
       输出：在线 Demo + 2min 视频 + 10 份反馈 + 数据采集启动
```

### 交接标准

| 交接 | 从 → 到 | 交接物 | 验收条件 |
|------|---------|--------|----------|
| PM → Designer | 需求 → 原型 | PRD + 用户故事 + 验收标准 | Designer 确认能出原型 |
| Designer → Dev | 原型 → 代码 | 交互原型 + 视觉稿 + 设计标注 | Dev 确认能实现 |
| Dev → QA | 代码 → 测试 | 代码 + 自审报告 | 代码能跑通核心流程 |
| QA → DevOps | 测试通过 → 部署 | Go 判定 + 测试报告 | QA 说 Go 才能部署 |
| DevOps → Ops | 部署上线 → 推广 | 在线 URL + 健康检查通过 | 站点可访问 |
| Ops → Data | 用户来了 → 采集数据 | 推广渠道数据 | 埋点数据开始流入 |
| Data → PM | 数据 → 决策 | 效果分析报告 | 报告有"所以呢"结论 |
| Data → CEO | 数据 → 校准 | 关键指标红/黄/绿 | CEO 根据数据做战略判断 |

## 一周一迭代节奏

```
周一（20min）         周三（10min）         周五（20min）
┌──────────┐        ┌──────────┐        ┌──────────────┐
│ 规划本周  │──────→│ 中期检查  │──────→│ 复盘+下周预划 │
│ @pm 主导  │       │ @pm 主导  │       │ @data+@pm 主导│
└──────────┘        └──────────┘        └──────────────┘
```

### 周一规划

- @pm 从 OKR 提取本周目标
- @designer + @dev 确认可行性
- @devops 确认部署通道畅通
- @data 确认数据采集就绪

### 周三检查

- @pm 检查进度
- 偏航 → 当天调整，不等到周五

### 周五复盘

- @data 出数据快报
- @pm 判断假设是否成立
- @ceo 做战略校准（如需要）
- 定下周方向

## 紧急流程

### P0 Bug 修复

```
@qa 发现 P0 → @dev 立即修复 → @qa 验证 → @devops 热部署
全程 < 1 小时
```

### 方向大转

```
@data 报告假设不成立 → @pm 提出新假设 → @ceo 批准 → 下一周立刻切方向
不犹豫，不停滞
```

### 机会窗口

```
外部出现大机会 → @ceo 判断是否值得切 → @pm 评估可行性 → 如果 3 天能出 Demo → 立刻启动快速出 Demo 流程
```
