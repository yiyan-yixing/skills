# Agent 效果评估体系

> 量化评估 Agent 产出质量，确保体系在持续改善而不是原地踏步。

## 评估维度

| 维度 | 指标 | 目标 | 采集方式 |
|------|------|------|----------|
| **PRD 质量** | 验收标准可量化率 | ≥ 90% | @data 审查 |
| **原型速度** | PRD 到可运行 Demo | ≤ 3 天 | 时间戳记录 |
| **代码质量** | 自审发现率 | ≥ 80% 的 bug 被 @dev 自审发现 | @qa 对比 |
| **测试覆盖** | 核心路径覆盖率 | ≥ 80% | @qa 测试报告 |
| **部署速度** | push 到上线 | ≤ 10 分钟 | @devops 计时 |
| **数据时效** | 上线到首次数据报告 | ≤ 7 天 | @data 时间戳 |
| **质疑有效性** | 质疑打回率 | 10-30%（太低=质疑走过场，太高=质量太差） | challenges.md 统计 |
| **迭代速度** | 一周迭代完成率 | ≥ 80% | current-sprint.md 统计 |

## 评估流程

```
每次迭代结束（周五复盘时）：
1. 协调者从 blackboard/ 和各 Agent 产出中提取指标
2. 填入本周评估表
3. 对比上周，标注趋势（↑/→/↓）
4. 低于目标的指标 → 下周改进项
```

## 评估报告模板

```markdown
# Sprint-<N> 评估报告

## 关键指标
| 指标 | 目标 | 本周 | 上周 | 趋势 | 备注 |
|------|------|------|------|------|------|

## 亮点
- ...

## 待改进
- ...

## 下周行动
- [ ] 行动 1
- [ ] 行动 2
```

## 产出存档

每个 Agent 的关键产出存档到 `agent-outputs/`：

```
agent-outputs/
├── sprint-1/
│   ├── pm-prd.md          # @pm 产出的 PRD
│   ├── architect-adr.md   # @architect 产出的 ADR
│   ├── designer-prototype/ # @designer 产出的原型
│   ├── dev-code/           # @dev 产出的代码（或 commit hash）
│   ├── qa-report.md        # @qa 测试报告
│   ├── data-report.md      # @data 效果报告
│   └── eval.json           # 本周评估指标
└── sprint-2/
    └── ...
```

## 反模式

- ❌ 只记录不分析——数据堆着不看 = 白记
- ❌ 指标太多看不过来——8 个维度足够
- ❌ 只看指标不看原因——数据下降时必须追因
- ❌ 把评估当惩罚——评估是为了改善，不是扣分
